# getFullPath 修复验证清单

**修复内容：** 改进 `findPathBetweenPoints()` 支持子路径提取

## 修复原理

之前 getFullPath() 失败因为：
- Dijkstra 返回的节点序列 ["B", "(47,34)", "(32,33)", "C"] 是图中的所有点
- 这些点来自 pathNetwork 中的某条路径的不同位置
- 旧的 findPathBetweenPoints() 只能找完整的 B→C 路径，找不到 B→(47,34) 的子路径

**新逻辑：**
1. 尝试找完整的 start→end 路径（原有）
2. 如果失败，为 pathNetwork 中每条路径的所有点生成标签（与图构建逻辑一致）
3. 精确匹配 startLabel 和 endLabel，提取中间的坐标子数组

## 故障排查

### 问题 1：仍然看到 `❌ 路径不完整`

**原因：** 子路径提取仍然失败  
**检查：**
1. 路径是否有足够的中间点？（否则都是关键点，不需要子路径提取）
2. 节点标签是否匹配？在日志中查看 Dijkstra 返回的标签
3. pathNetwork 中是否真的有这些标签对应的点？

**诊断命令（控制台）：**
```javascript
// 查看某条路径的标签序列
const path = Object.values(pathNetwork)[0];
const labels = path.map(pt => {
  for (const [kp, kpPt] of Object.entries(keyPoints)) {
    if (Math.sqrt(Math.pow(pt.x - kpPt.x, 2) + Math.pow(pt.y - kpPt.y, 2)) < 2) {
      return kp;
    }
  }
  return `(${Math.round(pt.x)},${Math.round(pt.y)})`;
});
console.log('路径标签：', labels);
```

### 问题 2：图中有节点但 Dijkstra 找不到路径

**原因：** 路径断开（例如 B→C 路径的起点不是 B，或不接近 B）  
**检查：**
1. 绘制 B→C 路径时，是否从 B 点开始？
2. B→C 路径的起点坐标是否接近 B 的坐标 (50,35)？

**诊断：**
```javascript
// 检查路径的起点和终点
Object.entries(pathNetwork).forEach(([key, path]) => {
  const start = path[0];
  const end = path[path.length-1];
  console.log(`${key}: (${start.x.toFixed(0)},${start.y.toFixed(0)}) → (${end.x.toFixed(0)},${end.y.toFixed(0)})`);
});
```

### 问题 3：getFullPath 返回空数组

**原因：** 某一段子路径找不到  
**检查日志：**
- 找 `[未找到]` 错误
- 检查失败的那一对标签是否真的在同一条路径上

## 预期行为

### 单条完整路径 A→C

```
路径：A (10,35) → (20,40) → (30,45) → C (30,30)

Dijkstra输出：["A", "(20,40)", "(30,45)", "C"]

getFullPath逻辑：
  A→(20,40)：找A→C路径，提取[0:2]
  (20,40)→(30,45)：找A→C路径，提取[1:3]
  (30,45)→C：找A→C路径，提取[2:4]

结果：完整坐标数组 [A, (20,40), (30,45), C]
```

### 多条连接的路径 A→B→C

```
路径1：A (10,35) → (25,35) → B (50,35)
路径2：B (50,35) → (40,32) → C (30,30)

Dijkstra输出：["A", "(25,35)", "B", "(40,32)", "C"]

getFullPath逻辑：
  A→(25,35)：在路径1中，[0:2]
  (25,35)→B：在路径1中，[1:3]
  B→(40,32)：在路径2中，[0:2]
  (40,32)→C：在路径2中，[1:3]

结果：跨路径的完整坐标数组
```
