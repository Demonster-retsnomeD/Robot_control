# 路径网络诊断与修复指南

## 当前状态

✅ **代码已就绪** (commit 6151c63)
- 机器人位置检测：支持关键点和路径中间位置
- 任务分配算法：直线距离选择（Euclidean）
- 图构建：支持真实路径的邻接关系

❌ **pathNetwork 未连接**
- 当前路径是孤立的分支，无法形成完整的图
- 示例：`"A-B"`, `"C-(37,30)"` 等各自独立
- 结果：Dijkstra 无法找到跨越多个路径的路由

---

## 问题根源

当你创建路径时，系统存储为 `pathNetwork[key] = [点数组]`。图构建时：
1. 扫描每条路径的起点和终点
2. 查找是否有关键点 A/B/C/D/E 在 2.0 像素内
3. 如果有，使用关键点标签作为图节点
4. 如果没有，创建伪标签 `"(x,y)"`

**问题：** 如果路径是这样创建的：
```
路径1: A → → → B  (key: "A-B")
路径2: C → → → (中间点)  (key: "C-(37,30)")  
路径3: E → → → D  (key: "E-D") 
```

图结构变成：
```
[A ↔ B]  （孤立）
[C ↔ (37,30)]  （孤立）
[E ↔ D]  （孤立）
```

没有边连接这些分量，所以无法从 B 到 C。

---

## 正确的路径创建方式

需要创建一条 **连续的链**：

```
A → 中间点1 → B → 中间点2 → C → 中间点3 → D → 中间点4 → E
```

### 分解为5条路径段：

1. **路径1: A → B**
   - 起点：A 点（10, 10）
   - 终点：B 点（50, 35）
   - 关键点检测：`startPt ≈ (10,10)` → "A", `endPt ≈ (50,35)` → "B"
   - 图节点：`"A" ↔ "B"`

2. **路径2: B → C**
   - 起点：B 点（50, 35）
   - 终点：C 点（30, 30）
   - 关键点检测：`startPt ≈ (50,35)` → "B", `endPt ≈ (30,30)` → "C"
   - 图节点：`"B" ↔ "C"`

3. **路径3: C → D**
   - 起点：C 点（30, 30）
   - 终点：D 点（70, 50）
   - 关键点检测：`startPt ≈ (30,30)` → "C", `endPt ≈ (70,50)` → "D"
   - 图节点：`"C" ↔ "D"`

4. **路径4: D → E**
   - 起点：D 点（70, 50）
   - 终点：E 点（80, 20）
   - 关键点检测：`startPt ≈ (70,50)` → "D", `endPt ≈ (80,20)` → "E"
   - 图节点：`"D" ↔ "E"`

5. **路径5: E → A**（可选，用于闭环）
   - 起点：E 点（80, 20）
   - 终点：A 点（10, 10）
   - 图节点：`"E" ↔ "A"`

### 最终图结构：

```
A ↔ B ↔ C ↔ D ↔ E ↔ A
```

现在任意两点间都能路由：
- A → C：A → B → C
- C → E：C → D → E
- E → B：E → A → B
- 等等

## 具体操作步骤

### 第一步：清空所有路径

打开 http://robot-sim-iota.vercel.app（或本地打开 path-network.html）

1. 进入 **编辑模式**（点"✏️ 编辑"按钮）
2. 点"🗑️ 删除模式"
3. 逐一删除所有已保存的路径（红色提示时点击路径段删除）
4. 验证：编辑状态栏显示"已保存: 0 条路径"

### 第二步：绘制 A→B 路径

1. 返回 **绘制模式**（编辑→绘制）
2. 点击 A 点（绿色圆，左上角）
3. 用鼠标绘制一条弯曲的路径到 B 点（橙色圆，右边）
4. 点"💾 保存"
5. 验证：起点标签 = "A"，终点标签 = "B"

### 第三步：绘制 B→C 路径

1. 点击 B 点（绿色圆，右边）
2. 绘制路径到 C 点（紫色圆，中间）
3. 点"💾 保存"
4. 验证：起点标签 = "B"，终点标签 = "C"

### 第四步：绘制 C→D 路径

1. 点击 C 点（紫色圆，中间）
2. 绘制路径到 D 点（假设位置 70,50）
3. 点"💾 保存"

### 第五步：绘制 D→E 路径

1. 点击 D 点（假设位置 70,50）
2. 绘制路径到 E 点（假设位置 80,20）
3. 点"💾 保存"

## 常见问题排查

### 问题1：`[任务C] ❌ 路径不完整（节点:["A","B"]）`
**原因：** 图中有 A-B 边，但没有 B-C 边
**解决：** 确认 B→C 路径已保存，且 B 点位置与前一路径的终点重合

### 问题2：`[Dijkstra] ❌ 无法到达C（从A出发）`
**原因：** C 不在图中的任何可达路径上
**解决：** 检查 C→D 或 D→C 路径是否已保存

### 问题3：机器人沿直线移动到 C，而不是沿路径
**原因：** 路径不存在，触发 fallback（直线）
**解决：** 检查 Dijkstra 日志，确保路径节点完整

## 关键点坐标（参考）

根据代码，关键点位置应该是：

```javascript
keyPoints = {
  A: { x: 10, y: 35 },   // 左边
  B: { x: 50, y: 35 },   // 右边
  C: { x: 30, y: 30 },   // 中间
  D: { x: 70, y: 50 },   // 右下
```

绘制路径时，确保起点和终点接近这些坐标。
